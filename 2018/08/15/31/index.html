线程取消

Posted on 2018-08-15

线程取消

　　所谓的线程取消，就是向线程发出一个请求，要求其立即退出，如线程正在执行一个运算，一旦线程检测到由错误发生，需要其他线程立即退出。

#include <pthread.h>
int pthread_cancel(pthread_t thread);

　　发出取消请求后，pthread_cancel()立即返回，不会等待目标线程的退出。

线程取消控制：取消状态
#include a<pthread.h>
int pthread_setcancelstate(int state,int *oldstate);
int ptread_setcanceltype(int type,int *oldtype);

　　 取消状态即为state指定的值。
　　 具体参见：TLPI P553
　　
　　 如果将取消性状态和类型设置为启动和延迟，仅当线程到达某个取消点时，取消请求才会起作用。
　　 函数pthread_testcancel()可以产生一个取消点，线程如果已经处于挂起状态，那么只要调用该函数，线程就会随之终止。

#Include <pthread.h>
void pthread_testcancel(void);//可以用于线程可取消行检测
清理函数

　　为避免线程取消导致共享变量和Pthreads对象(如互斥量)置于一种不一致的状态，可能导致进程中的其他线程产生错误结果，死锁，以及程序崩溃，每个线程都有一个线程函数栈，当线程取消时，自顶而下的执行线程清理函数，当执行完线程终止函数时，线程终止。

#incldue <pthread.h>
void pthread_cleanup_push(void (*routine)(void *),void *arg);
void pthread_cleanup_pop(int execute);

pthread\_cleanup\_push()会将参数routine所包含的函数地址添加到调用线程的清理函数栈定顶，参数routine是一个函数指针。arg作为调用清理函数的参数。
如果线程顺利执行完代码而未取消，那么就不需要进行清理，pthread\_cleanup\_pop()中的execute非零，那么无论如何都会执行清理函数。
异步取消

　　设置线程为异步取消时，即取消类型为PTHREAD_CANCEL_ASYNCHRONOUS,可以在任何时点将其取消，取消动作不会拖延到下一个取消点。
　　作为一般性原则，可异步取消的线程不应该分配任何资源，也不能获取互斥量和锁。导致Pthreads()函数的大部分，如pthread_cancel()，pthread_setcancelstate()以及pthread_canceltype()，就是必须确保在可异步取消的线程中大量调用他们，例如取消执行计算密集型循环的线程。异步取消较少使用。
