线程

Posted on 2018-08-12

线程

　　与进程类似，线程是允许应用程序并发执行多个任务的一种机制。一个进程可以包含多个线程。同一个程序中的所有线程均会独立的执行相同程序，且共享同一份全局内存区域，包括初始化数据段，未初始化数据段，以及堆内存段。

问题

　　进程间的信息难以共享，由于除去只读代码段之外，父子进程并未共享内存，因此采用IPC在进程间进行信息交换。
　　调用fork()相对代价较高，即使使用copy-on-write技术，仍然需要复制诸如内存页表，文件描述符表之类的多种进程属性。
　　线程解决了以上两个问题。

  • 线程之间只需要将数据复制到共享(全局或堆)变量中即可。
  • 创建线程要比进程快快很多，通过系统调用clone()来实现
创建线程

　　启动程序时，产生的线程只有单挑线程，称之为初始或主线程。

#include <pthread.h>
int pthread_create(pthread_t *thread,const pthread_attr_t *attr,void *(*start)(void *),void *arg);

　　新线程通过调用带有参数arg的函数start而开始执行，调用pthread_create()的线程会继续执行该调用之后的语句。

终止线程
  • 线程start 函数执行return 语句后并返回指定值。
  • 线程调用pthread_exit()。
  • 调用pthread_cancel()取消线程。
  • 任意线程调用了exit()，或者主线程执行了return 语句，都会导致进程中的所有线程终止。

　　pthread_exit()函数将终止调用进程，且其返回值可由另一线程通过调用pthread_join()来获取。

#include <pthread.h>
void pthread_exit(void *retval);

　　调用pthread_exit()相当于在线程的start函数中执行return ,不同在于，可以在线程start函数所调用的任意函数中调用pthread_exit()。
　　如果主线程调用了pthread_exit()，而非调用exit()或是执行return语句，那么其他线程将继续运行。

线程ID

　　线程ID会返回给pthread_create()的调用者，一个线程可以通过调用pthread_self()来获取自身的线程ID。
　　pthread_equal()用于比较线程ID是否相等。

连接已终止的线程

　　pthread_join()等待由thread标识的线程终止（如果线程已经终止，线程会立即返回），这种操作称为连接。
　　如果向pthread_join()传入一个之前已经连接过的线程ID，将导致无法预知的行为。
　　若线程并未分离，则必须使用pthread_join()来进行连接，如果未能连接，将产生僵尸进程。
　　pthread_join()执行的功能类似与针对进程waitpid()调用，区别在于：

  • 线程之间的关系时对等的，进程中的任意线程均可以调用pthread_join()与g该进程的任何其他线程连接起来。只有父进程可以通过wait()来对子进程进行回收。
  • 无法“连接任意线程”，也不能以非阻塞方式进行连接。
线程的分离

　　默认状态下线程是可连接的(joinable),即线程退出时，其他线程可以通过pthread_join()来获取其状态。

#include <pthread.h>
int pthread_detach(pthread_t thread);

　　可以调用pthread_detach()，将线程标记处于分离状态，线程可以自行分离。
　　一旦线程处于分离状态，就不能再使用pthread_join()来获取其状态，也无法使其返回可连接状态。
　　如果其他线程调用了exit()，或是主线程执行return语句时，包括遭到分离的线程，进程的所有线程都会终止。pthread_detach()只控制线程终止之后发生的事情，而非何时或如何终止线程。
