Θ Notation

　　When the upper and lower bounds are the same within a constant factor,we indicate this by using Θ notation.

Simplifying Rules

　　1.if f(n) is in O(n) and g(n) is in O(n),then f(n) is in O(h(n)).
　　2.if f(n) is in O(kg(n)) for any constant k>0,then f(n) is in O(n).
　　3.if f1(n) is in O(g1(n)) and f2(n) is in O(g2(n)) ,then f1(n) + f2(n) is in O(max(g1(n),g2(n))).
　　4.if f1(n) is in O(g1(n)) and f2(n) is in O(g2(n)) ,then f1(n)f2(n) is in O(g1(n2(n)).

　　The problem of sorting is Θ(nlogn) in the worse case.
　　The upper bound for an algorithm is not the same as the worse case for that algorithm for a given input of size n. what is being bounded is not the running time (which you can determine for a given value of n), but rather the growth rate for the running time(which can only be determined by the range of values for n). 　　 　　 　　

the comparison of Array-based and Linked Stacks

　　All operations for the array-based and linked stack implementations take constant time.
　　The Array-based stack must declare a fixed-size array initially,and some of that space is wasted whenever the stack is not full.
　　The linked stack can shrink and grow but requires the overhead of a link field for every element.

the number of leaves in a non-empty full binary tree is one more than the number of internal nodes.
