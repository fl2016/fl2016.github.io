进程创建和程序执行深入

进程记账

　　打开进程记账功能后，内核会在每个进程终止时将一条记账信息写入系统级的进程记账文件，这个账单包含进程的终止状态和进程消耗CPU时间。

打开和关闭进程记账功能

　　特权进程可以利用系统调用acct()来打开和关闭进程记账功能。

#define /_BSD/_SOURCE
#include <unistd.h>
int acct(const char *acctfile);

　　acctfile中指定一个现有常规文件的路径名，记账文件通常的路径名为/var/log/pact或/usr/account/pacct.关闭记账功能只需将acctfile置位NULL

系统调用

　　类似于fork()和vfork()，linux特有的系统调用也能创建一个新进程，，clone()主要用于线程库的实现。

#define _GNU_SOURCE
#include <sched.h>
int clone(int (*func)(void*),void *child_stack,int flags,void *func_arg,...);

　　与fork()不同的是，克隆生成的子进程继续运行时不以调用处为起点，转而去调用func所指定的函数，func又称子函数，子函数对该参数的含义可以自由解读，可以作为整形值，或者视为指向结构的指针（因为克隆产生的子进程对调用进程的内存即可获取，也可共享）。
　　一个线程组内的每个线程都有唯一的线程标识符TID。
　　线程可以调用gettid（）获取自身的线程ID。
　　线程ID在整个系统中是唯一的，只有线程担当进程中线程组首线程时线程ID和进程ID相同。
　　线程组中的所有线程用于同一个父进程ID。
　　线程与进程不同，不能用wait()等待，必须调用pthread_join()来加入。
