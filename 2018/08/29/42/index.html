POSIX 消息队列

打开，关闭和断开链接消息队列
打开一个消息队列
#include <fcntl.h>
#include <sys/stat.h>
#include <mqueue.h>
mqd_t mq_open(const char*name,int oflag,...);

　　name参数标识出了消息队列，oflag参数是一个位掩码。

fork(),exec()以及进程终止对消息队列描述符的影响

　　在fork()中子进程通常会接收其父进程的消息队列描述符的副本，并且这些描述符会引用同样的打开着的消息队列描述，子进程不会继承其父进程的任何消息通知注册。
　　当一个进程执行了一个exec()或者终止时，所有其打开的消息队列描述符会被关闭。关闭消息队列描述符的结果是进程在相应队列上的消息通知注册会被注销。

关闭一个消息队列

　　mq_close()函数关闭消息队列描述符mqdes。

#include <mqueue.h>
int mq_close(mqd_t mqdes);

　　如果调用进程已经通过mqdes在消息队列上注册了消息通知，那么通知注册会自动被删除，并且另一个进程可以随后向该队列注册消息通知。

删除一个消息队列

　　mq_unlink()函数删除了通过name标识的消息队列，并将队列标记为所有进程使用完该队列之后销毁该队列(这意味着会立即删除，前提是所有打开该队列的进程都已经关闭了该队列)。

#include <mqueue.h>
int mq\_unlink(const char *name);
描述符和消息队列之间的关系

　　消息队列描述符和打开的消息队列之间的关系与文件描述符和打开的文件描述符的关系类似，消息队列描述符是一个进程级别的句柄，它引用了系统层面的打开着的消息队列描述表中的一个条目，而该条目则引用了一个消息队列对象。

获取消息队列特性

　　mq_getattr()函数返回一个与描述符相关的消息队列描述和消息队列的相关信息的mq_attr结构。

#include <mqueue.h>
int mq_getattr(mqd_t mqdes,struct mq_attr *attr);
修改消息队列特性

　　mq_setattr()函数设置与消息队列描述符mqdes相关联的消息队列的特性，并可选的返回与消息队列相关的消息。

int mq_setattr(mqd_t mqdes,const struct mq_attr *newattr,struct mq_attr *oldattr);

　　mq_setattr()函数执行以下任务。

  • 它使用newattr指向的mq_attr结构中的mq_flags字段来修改与描述符mqdes相关联的消息队列描述的标记。
  • 如果oldattr不为NULL，那么就返回一个之前的消息队列描述标记和消息队列特性的mq_attr结构。
交换信息
发送信息

　　mq_send()函数将位于msg_ptr指向的缓冲区中的消息添加到队列中。

#include <mqueue.h>
int mq_send(mqd_t mqdes,const char *msg_ptr,size_t msg_len,unsigned int msg_prio);

　　msg_len 参数制定了msg_ptr指向的消息的长度，其值必须小于或者等于队列的mq_msgsize特性，否则msg_send()就会返回EMSGSIZE错误，长度为0的消息是允许的。
　　每个消息都有一个非负整数表示的优先级，它通过msg_prio参数指定。消息在队列中是按照优先级倒序排列的，即0表示的优先级最低，当一个消息被添加到队列中，它会被放置在队列中具有相同优先级的所有消息之后。如果一个应用程序无需使用优先级，那么只需要将msg_prio指定为0。

接收消息

　　mq_receive()函数从mqdes引用的消息队列中删除一条优先级最高，存在时间最长的消息并将删除的消息放置在msg_ptr指向的缓冲区。

#include <mqueue.h>
ssize_t mq_receive(mqd_t mqdes,char *msg_ptr,size_t msg_len,unsigned int *msg_prio);

　　调用者使用msg_len参数来指定msg_ptr指向的缓冲区中的可用字节数。
　　不管消息的实际大小是什么，msg_len(即msg_ptr指向的缓冲区的大小)必须大于等于队列中的mq_msgsize特性，否则mq_receive()就会失败并返回EMSGSIZE错误。如果不清楚一个队列mq_msgsize特性的值，那么可以使用mq_getattr()来获取这个值。

在发送和接收消息时设置超时时间

　　mq_timedsend()和mq_timedreceive()函数与mq_send()和mq_receive()几乎是一样的。唯一的区别在于如果操作无法立即执行，并且该消息队列描述上O_NONBLOCK标记不起作用，那么abs_timeout参数就会为调用阻塞的时间指定一个上限。

#include <mqueue.h>
#include <time.h>
int mq_timedsend(mqd_t mqdes,const char *msg_ptr,size_t msg_len,unsigned int msg_prio,const struct timespec *abs_timeout);
ssize_t mq_timedreceive(mqd_t mqdes,char *msg_ptr,size_t msg_len,unsigned int *msg_prio,const struct timespec *abs_timeout);

　　abs_timeout参数是一个timespec结构，它将超时事件设置为自新纪元到现在的一个绝对值，其单位为秒数和纳秒数。

消息通知

　　该功能可无需执行一个阻塞的调用或者将消息队列描述符标记为非阻塞并在队列上定期执行 mq_receive()调用了，因为一个进程能够请求消息到达通知。

#include <mqueue.h>
int mq_notify(mqd_t mqdes,const struct sigevent *notification);

　　notification参数指定了进程接收通知的机制。

  • 在任何一个时刻都只有一个进程(“注册进程”)能够向一个特定的消息队列注册接收通知，如果一个消息队列上已经存在注册进程了，那么后续在该队列上的注册请求将失败。
  • 只有一个新消息进入之前为空的队列时注册进程才会收到通知。
  • 当注册进程发送一个通知后就会删除注册信息，之后任何进程就能够向队列注册接收通知了。
  • 注册进程只有当前不存在在其他在该队列上调用mq_receive()而发生阻塞的进程时才会收到通知，如果其他进程在mq_receive()调用中被阻塞了，那么该进程就会读取消息，注册进程会保持注册状态。
  • 一个进程通过mq_notify()时传入一个NULL的notification参数来撤销自己在消息通知上的注册消息。
    　　linux实现上，消息队列描述符实际上就是一个文件描述符，在linux中,每个进程以及系统所能打开的文件描述符的限制实际上会用于文件描述符和消息队列描述符之和。
