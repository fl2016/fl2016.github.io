POSIX 信号量

打开一个命名信号量

　　sem_open()函数创建和打开一个新的命名信号量或打开一个既有信号量。

#include <fcntl.h>
#include <sys/stat.h>
#include <semaphone.h>
set_t *sem_open(const char *name,int oflag,...);

　　name参数标识出了信号量，oflags参数是一个位掩码，它确定了是打开一个信号量还是创建一个新的信号量。
　　不管是创建一个新信号量还是打开一个既有信号量，sem_open()都会返回一个指向一个sem_t 的指针，而在后续调用中则可以通过这个指针操纵这个信号量。

关闭一个信号量

　　当一个进程打开一个命名信号量时，系统会记录进程和信号量之间的关联关系，sem_close()函数会终止这种关联关系(即关闭信号量)，释放系统为该进程关联到该信号之上的所有资源，并递减引用该信号量的进程数。

#include <semaphore.h>
int sem_close(sem_t *sem);

　　打开的命名信号量子在进程终止或者进程执行了一个exec()时会自动被关闭。关闭一个信号量并不会删除这个信号量，删除信号量要用sem_unlink()。

删除一个命名信号量

　　sem_unlink()函数删除通过name标识的信号量并将信号量标记为一旦所有进程都使用完这个信号量时就销毁该信号量。

#include <semaphore.h>
int sem_unlink(const char *name);
信号量操作
#include <semaphore.h>
int sem_wait(sem_t *sem);

　　如果信号量的当前值大于0，那么sem_wait()会立即返回，如果信号量的当前值等于0，那么sem_wait()会阻塞直到信号量的值大于0为止，当信号量大于0时该信号量值就被递减并且sem_wait()就会立即返回。

信号量的发布
＃include <semaphore.h>
int sem_post(sem_t *sem);

　　如果在sem_post调用之前信号量的值为0，并且其他某个进程或线程正在因等待递减这个信号量而阻塞，那么该进程会被唤醒，它的sem_wait()调用会继续往前执行来递减这个信号量。

获取信号量的当前值
#include <semaphore.h>
int sem_getvalue(sem_t *sem,int sval);

　　将sem引用的信号量的当前值通过sval指向的int变量返回。

未命名信号量

　　未命名信号量(也称基于内存的信号量)是类型为set_t并存储在应用程序分配的内存中的信号量，通过将这个信号量放在几个进程或者线程共享的内存中就能够使得这个信号量对这些进程或者线程可用。
　　操作未命名信号量所使用的函数与操作命名信号量使用的函数是一样的，此外：
　　　　　１．sem_init()函数对一个信号量进行初始化并通知该信号量会在进程间共享还是会在单个进程中的线程间共享。
　　　　　２．sem_destory(sem)函数会销毁一个信号量。

  • 在线程间共享的信号量不需要名字，将一个未命名信号量作为一个共享变量(全局或堆上)自动会使之对所有线程可访问。
  • 在相关进程间共享的信号量不需要名字。
  • 如果正在构建的是一个动态数据结构(如二叉树),并且其中每一项都需要一个关联的信号量，简单实现可以是分配一个未命名信号量。
初始化一个未命名信号量

　　sem_init()函数使用value中指定的值来对sem指向的未命名信号量进行初始化。

#include <semaphore.h>
int sem_init(sem_t *sem,int pshared,unsigned int value);

　　1，如果pshared等于0，那么信号量将会在调用进程中的线程间共享，sem通常会被指定为一个全局变量的地址或者分配在一个堆上的变量的地址，线程共享的信号量具备进程持久性，他在进程终止时会被销毁。
　　2.如果pshared不等于0，那么信号量将在进程间共享，在这种情况下，sem必须是共享内存区域(一个POSIX共享内存对象，一个使用mmap()创建的共享映射或者一个system V内存段)中某个位置的地址。信号量的持久性和它所处共享内存的持久性是一样的。

销毁一个未命名信号量

　　sem_destory()函数将销毁信号量sem，其中sem必须是一个之前使用sem_init()进行初始化的未命名信号量，只有在不存在进程或线程在等待一个信号量时才能安全销毁这个信号量。

#include <semaphore.h>
int sem_destory(sem_t *sem);

　　当使用sem_destory()销毁了一个未命名信号量之后就能够使用sem_init()来重新初始化这个信号量了。
　　一个未命名的信号量应该在底层的内存被释放之前被销毁。
