SOCKET介绍

Posted on 2018-09-17

SOCKET:介绍

　　socket是一种IPC方法，其通信方式如下：
　　　　1.各个应用程序创建了一个socket，socket是一个允许应用程序通信的设备，两个应用程序都需要用到它。
　　　　2.服务器将自己的socket绑定到一个众所周知的地址上能够是使的客户端定位到它的位置。

fd = socket(domain,type,protocol);
通信domain

　　socket存在于一个domain中，它确定：

  • 识别出了一个socket方法。
  • 通信范围

　　三种domain,UNIX，IPv4,IPv6。

socket类型

　　每种socket实现都至少提供两种socket:流和数据报，流是面向连接的，可靠传送，没有消息边界，数据报是非面向连接的，由消息边界。
　　流socket（SOCK_STREAM)提供了一个可靠的双向的字节流通道。
　　流socket的正常工作需要一对相互连接的socket，因此流socket通常是称为面向连接的，一个流socket只能与一个对等socket进行连接。
　　数据报socekt在使用时无需与另一个socket连接。
　　在internet domain中，数据报socket使用了数据报协议，流socket通常使用了TCP协议。

socket 系统调用
  • socket系统调用创建一个新socket。
  • bind系统调用将一个socket绑定到一个地址上，通常服务器将socket绑定到一个众所周知的地址绑定上以使得客户端能够定位到该socet。
  • listen()系统调用允许一个流socket接受来自其他socket的接入连接。
  • accept()系统调用允许在一个监听流socket上接受一个来自对等应用程序的连接。
  • connect()系统调用建立与另一个socket之间的连接。
  • socket I/O可以使用传统的read()和write()，或者使用socket特有的系统调用(如send()，sendto()，recv()，recvfrom())来完成，默认情况下，这些系统调用在I/O操作无法完成时会被阻塞，通过使用fcntl()F_SETFL操作来启用O_NONBLOCK打开文件状态标记可以执行非阻塞I/O。
创建一个socket
#include <sys/socket.h>
int socket(int domain,int type,int protocol);

　　domain参数制定了socket通信的domain，type参数指定了socket类型，type可以是SOCK_STREAM或者SOCK_DGRAM。
　　protocol参数在本书描述的socket类型中总是会被指定为0.

将socket绑定到地址

　　bind()系统调用将一个socket绑定到一个地址上。

#incldue <sys/socket.h>
int bind(int sockfd,const struct sockaddr *addr,socklen_t addlen);

　　sockfd参数是一个sock()调用中获得文件描述符，addr参数是一个指针，指向了一个指定该socket绑定到的地址的结构，传入的这个参数的结构的类型取决于socket domain,addrlen指定了地址结构的大小。

通用的socket地址结构：struct sockaddr

　　UNIX domain socket 使用路径名，Internet domain socket中使用IP地址和端口号。

struct sockaddr{
    sa_family_t sa_family;
    char sa_data[14];
    };

　　这个结构是所有domain特定的地址结构的模板，其中每个地址结构均与sockaddr结构中sa_family字段的family字段打头。

流socket

　　主动socket和被动socket：
　　　　1.默认情况下，使用socket创建的socket是主动的，一个主动的socket可以在connect系统调用中来创建贵到被动socket连接，这种行为称为执行一个主动的打开。
　　　　2.一个被动socket（也称为监听socket）是一个通过listen()以被标记成允许接入连接的socket。
　　　　大多数情况下，服务器执行被动式打开，客户端执行主动式打开。

监听接入连接：listen()

　　listen()系统调用将文件描述符sockfd引用的流标记为被动，这个socket后面会被从来接受来自其他主动socket的连接。

#include <sys/socket.h>
int listen(int sockfd,int backlog);

　　无法在一个已连接的socket（即已成功执行connect的socket或由accept()调用返回的socket）上执行listen。
　　内核必须记录所有未决的连接请求的相关信息，这样后续accept()就能够处理这些请求了，backlg参数允许限制这种未决的连接的数量。

接受连接:accept()

　　accept()系统调用在文件描述符sockfd引用的监听流上接受一个接入连接，如果调用accept()不存在未决的连接，那么调用会阻塞到有连接请求到达为止。

#include <sys/socket.h>
int accept(int sockfd,struct sockaddr *addr,socket_t *addrlen);

　　理解accept()的关键点是它会创建一个新socket，并且正是这个新socket会与执行connect()的对等socket进行连接。
　　剩余参数返回对端的socket的地址，addr参数指向了一个用来返回socket地址的结构，这个参数的类型取决于socket domain.
　　addrlen参数是一个值结果参数，它指向了一个整数，在调用之前将这个整数初始化为指向addr的缓冲区的空间的大小。这样内核就知道有多少空间可以返回socket地址了。accept()返回之后，这个整数就可以设置成复制进缓冲区的数据的字节数。
