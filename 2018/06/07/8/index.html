用户和组

密码文件 /etc/passwd

　　如果启动了shadow密码，则密码字段中通常会包含x，如果/etc/passwd中密码字段为空，则该账户登录时不需要密码。 （即使启动了shadow密码也是这样）
　　用户ID（UID）:如果该ID为0，相应账户有特权权限， 该账户只有一个，即root。
　　组ID(GID):用户首选属性组的ID.

shadow密码文件 /etc/shadow

　　为防范黑客用碰撞破解密码信息，shadow文件应运而生，其理念是所有非敏感信息放在人人可读的密码文件中，经过加密处理的密码则由shadow密码文件单独维护，只要具有特权的用户可读。

组文件 /etc/group

　　系统中的每个组在/etc/group中都有一条记录：users:x:100:memberlist。

获取用户和组信息

#include <pwd.h> 
struct passwd *getpwnam(const char *name);
struct passwd *getpwuid(uid_t uid);//从密码文件中获取记录，成功则返回指针，失败返回NULL。

　　为name提供一个登录名，getpwnam()和getpwuid()函数就会返回一个指针，指向如下的结构,换言之，两者返回结果相同： 

struct passwd{
    char *pw_name;//登录用户名
    char *pw_passwd;//加密密码
    uid_t pw_uid;//用户ID
    gid_t pw_gid;//组ID
    char *pw_gecos;//用户评价
    char * pw_dir；//初始工作目录
    char *pw_shell;// 登录shell
    };

　　仅在未启用shadow密码情况下，pw_passwd才会包含有效信息，可通过成功调用getpwnam()之后，紧接着调用getspnam()，并观察后者是否能为同一用户名返回条shadow密码记录。

从组文件获取记录

#include <grp.h>
struct group *getgrnam(const char *name);
struct group *getgrgid(gid_t gid)；//成功则返回指针，失败则返回NULL

扫描密码文件和组文件中的记录

#include <pwd.h>
struct passwd *getpwent(void);
void setpwent(void);
void endpwent(void);

　　getpwent()可以顺序扫描密码文件，逐条返回记录，当不在有记录或者出错时，返回NULL,getpwent()一经调用，会自动打开文件，处理完毕后，用endpwent()将其关闭。

从shadow文件中获取记录

#include <shadow.h>
struct spwd *getspnam(const char *name);//成功返回指针，没有找到或错误返回NULL  
struct spwd *getspent(void);//成功则返回指针，到达流的尾部或者错误返回NULL

数据加密和用户认证

　　UNIX采用单向加密算法进行加密，即由加密密码的加密形式无法还原出原始密码，验证候选密码的唯一方法是使用同一算法进行加密，并将结果与存储在shadow中的密码进行匹配。 

#include <unistd.h>
char *crypt(const char *key,const char *salt);

　　crypt（）算法接受一个最长8字符的密钥，施之一种数据加密算法的一种变体(DES),salt参数指向一个两字符的字符串，用来扰动DES算法，使得加密算法更加难以破解，返回一个指针，指向一个长度为13字符的字符串。
　　salt是两个字符，可以使加密算法产生64*64种变化，在linux中使用crypt()，在编译程序时需要开启-crypt选项，以便在编译时链接crypt库
