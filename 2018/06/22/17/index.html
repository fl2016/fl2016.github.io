Posted on 2018-06-22

访问控制列表

　　一个ACL由一系列的ACL记录（简称ACE组成），每个记录都针对单个用户或用户组定义了对文件的访问权限。

ACL记录

　　每个ACE由3部分组成。

　　　　1.标记类型：表示该记录作用于一个用户，组，还是其他类别用户。
　　　　2.标记限定符：标识特定的用户或组。
　　　　3.权限集合：本字段包含所授予的权限信息。
　　ACL_USER_OBJ
　　 　　带有该标记的ＡＣＥ记录了授予文件属主的权限，每个ACL只能包含一个该类型标记的记录，该记录与传统的文件属主权限相对应。
　　ACL_ USER
　　　 　 携带该值的ACE记录了授予某用户的权限，一个ACL可包含零条或多条此标记类型的记录，但针对一个特定用户最多只能定义一条此类记录。
　　ACL_GROUP_OBJ
　　　　　包含该值的ACE记录了授予文件组的权限，每个ACL只会包含一条此标记类型的文件记录。除非ACL还包含类型为“ACL_MASK”的记录，否则此记录对应于传统的文件组权限。
　　ACL_MASK
　　 　　　包含了该值的ACE记录了可以由以上各型ACE所能授予的最高权限，一个ACL最多只能包含一条标记类型为ACL_MASK的ACE,假如ACL含有标记类型ACL_USER或ACL_GROUP的记录，那么就必须包含条ACL_MASK型的ACE。
　　ACL_OTHER
　　　　　对于不匹配任何其他ACE的用户，由包含该值的ACE授予权限，每个ACL只能包含一条标记类型为ACL_OTHER的ACE,该记录 对应于传统的文件其他权限。

最小化和扩展ACL

　　最小化ACL对应于传统的文件权限集合，恰好由ACL_USER_OBJ,ACL_GROUP,ACL_OTHER组成，扩展ACL是指除此之外，还包含标记类型为ACL_USER,ACL_GROUP,ACL_MASK的记录。

ACL权限检查算法

　　与传统的文件权限模型相比，对具有ACL的文献进行检查时，按以下顺序执行，直到某一标准得到匹配。
　　1.若进程具有特权，则拥有所有的访问权限，执行某文件时，仅当将可执行权限通过至少一条ACL记录授予该文件时，系统才会将特权级进程授予该权限。
　　2.若某一进程的有效用户ID匹配文件的属主（用户ID时），则授予该进程的标记类型为ACL_USER_OBJ的ACE指定权限，严格来讲，ACL检测使用的是进程的文件系统ID,而非其有效用户ID。
　　3.若进程的有效用户ID与某一ACL_USER类型记录的标记限定符相匹配，则授予该进程此记录所指定权限与ACL_MASK记录值相与的结果。
　　4.若进程的组ID即进程的有效组ID或者任一辅助组ID匹配于文件组（对应标记类型为ACL_GROUP_OBJ的ACE，或者任一ACL_GROUP型记录的标记限定符，则会依次进行以下检查，直至发现匹配项：
　　　　a.若进程的组ID之一匹配于文件组，且标记类型为ACL_GROUP_OBJ的ACE授予了所请求的权限，则会根据此记录来判定对文件的访问权限，如果ACL中还包含了标记类型为ACL_MASK的ACE,那么该文件的访问权限将是两记录权限相与之后的结果。
　　　　b.若进程的组ID之一匹配于文件所辖ACL_GROUP型ACE的标记限定符，且该ACE授予了所请求的权限，那么会根据磁记录来判定对文件的访问权限， 且该ACE授予所请求的权限，那么会根据此记录来判定对文件的访问权限，如果ACL中包含了ACL_MASK型的ACE,那么对文件的访问权限应是两记录权限相与的结果。
　　　　c.否则拒绝对该文件的访问。
　　5.否则，将以ACL_OTHER型ACE所记录的权限授予进程。

ACL的长短文本格式

　　执行setfacl和getfacl命令，或是使用某些ACL库函数操纵ACL时，需要指明ACL的两种文本表现格式：
　　1.长文本形式的ACL:每行都包含一条ACL,还可以包含注释，注释需要以“#”开始，直到行尾结束，getfacl命令的输出会以长文本格式显示ACL,getfacl命令的-M acl-file 选项从指定文件中提取长文本格式的ACL格式定义。
　　2.短文本格式：包含一系列以”，“分割的ACE。
　　每个ACE都是以下格式：

tag-type :[tag-qualifier]:permissions

　　标记类型之后的标记限定符为可选项，采用名称或者数字ID来标识用户或组，仅当标记类型为ACL_USER和ACL_GROUP时，才允许标记限定符的存在。

ACL_mask型的ACE和ACL组分类

　　如果一个ACL包含了标记类型为ACL_USER或ACL_GROUP型的ACE,那么一定会包含标记类型为ACL_MASK的ACE，否则，ACL_MASK为可选项。
　　对于ACL_MASK类型的ACE，其作用是在于所谓组分类中的ACE所能授予权限的上限，组分类是指在ACL中，由所有标记类型为ACL_USER,ACL_GROUP以及ACL_GROUP_OBJ的ACE所组成的集合。
　　提供标记类型为ACL_MASK的ACE，其目的在于即使运行并无ACL概念的程序也能保障其行为的一致性。
　　对于对ACL一无所知的应用程序而言，这意味着”除了文件属主外，不允许其他任何用户访问“。ACL_MASK，这一机制在实现传统意义上的chmod()操作的同时，也无损于由具有ACL概念的应用所确立的文件权限语义，当ACL包含标记类型为ACL_MASK的ACE时：
　　　1.调用chmod()对传统组权限做的变更，会改变对ACL_MASK(而非ACL_GROUP_OBJ)标记类型的设置。
　　　2.调用stat()，在st_mode 字段的组权位中会返回ACL_MASK权限（而非ACL_GROUP_OBJ权限）
　　　ACL_MASK型记录的出现保护了ACL信息，ACL的优先级要高于对文件组权限的传统操作，也就是说，如果修改了ACL_MASK权限，但是文件组权限没有，则还是无法获取相应的权限

getfacl和setfactl命令
setfacl -m u:guanyx2017:rx,g:guanyx2017:x myfile //设置文件权限掩码
getfacl --omit-header myfile // 查看文件权限掩码

　　带-m的setfacl命令可以修改现有ACE,或者当给定标记类型和限定符的ACE不在时，会追加新的ACE,setfacl命令还可以使用-R选项，将指定的ACL递归应用于目录中的所有文件。 

setfacl -m m::x myfile//用于禁用文件权限掩码中除了执行权限外的其他所有权限  
setfacl -x 用来从ACL中删除记录

默认ACL与文件创建

　　当进程访问与该ACL相关的文件时，将使用访问型ACL来判定进程对文件的访问权限，针对目录，可创建第二种ACL,默认型ACL。
　　访问目录时，默认型ACL并不判定所授予的权限，默认型ACL的存在与否决定了在目录下所创建文件和子目录的ACL和权限。（默认型ACL存储于名为system.posix_acl_default的扩展属性中）
　　查看和设置与目录有关的默认型ACL，需要执行带有-d选项的getfacl和setfacl命令：

setfacl -d -m u::rwx,u:guanyx2017:rx,g::rx,g:guanyx2017:rwx,o::- sub
getfacl -d --omit-header sub

　　带有 -k选项的setfacl命令，可删除对目录而设的默认型ACL。
　　1.新建于目录下的子目录会继承目录的默认ACL。
　　2.新建于目录下的文件或者子目录会将该目录的默认型ACL继承为其访问型ACL,与传统文献权限位相对应的ACL记录将和创建文件和子目录时调用系统调用（open(),mkdir()等）的mode参数相与，所谓的对应的ACL记录是指：
　　　　a.L_USER_OBJ
　　　　b.ACL_MASK，不包含ACL_MASK,则为ACL_GROUP_OBJ
　　　　c.ACL_OTHER
　　一旦目录拥有默认型ACL，对于新建于目录下的文件来说，进程的umask()并不参与判定文件访问型ACL中所记录的权限。

ACL在实现方面的限制

　　对与ext2,ext3,ext4文件系统，某文件所含ACL记录的总和要受制于以下要求，该文件的扩展属性的所有名称和值所占字节必须位于同意逻辑磁盘块之内，每条ACL记录占8字节。
　　对于XFS文件系统，每一ACL记录的上限值为25条。
　　程序要使用ACL API，应包含<sys/acl.h> ，要与libacl链接，编译此类程序需要带有-lacl选项。
　　　　1.一个ALC包含一条或者多条ACL记录。
　　　　2.每个记录均包含一条标记类型，以标记限定符，以及一权限集合。

将文件的ACL读入内存

　　acl_get_file()函数用来获取由（pathname所标识）文件的ACL副本。 

acl_t acl;
acl = acl\_get\_file(pathname,type);

　　取决于参数type的值，可用ACL_TYPE_ACCESS获取访问型ACL的值，可用ACL_TYPE_DEFAULT获取默认型ACL的值。
　　句柄指代一对象或者数据结构，句柄的表现形式由API实现决定，可以是指针，数组索引，hash键

从内存ACL中获取记录

　　acl_get_entry()函数会返回以句柄，指向内存ACL（由函数的acl参数指代)中的记录之一，句柄的返回位置由函数的最后一个参数决定。 

acl_entry_t entry;
status = acl_get_entry(acl,entry_id,&entry);

　　首次调用时，将entry_id制定为ACL_FIRST_ENTRY，返回指向ACL中的首条句柄，随后调用中将其指定为ACL_NEXT_ENTRY，如此就可以遍历ACL中的所有记录。
　　若成功获取到一条ACL,函数返回-1，否则，返回0。

获取并修改ACL记录中的属性

　　函数acl_get_tag_type()和acl_set_tag_type()分别用来获取和修改ACL中的标记类型。 

acl_tag_t tag_type;
status = acl_get_tag_type(entry,&tag_type);
status = acl_set_tag_type(entry,tag_type);

　　tag_type的类型为acl_type_t整形，取值可为ACL_USER_OBJ,ACL_USER,ACL_GROUP_OBJ,ACL_OTHER或ACL_MASK之一。
　　acl_getr_qualifier()和acl_get_qualifier()可分别用来获取和修改由entry参数制定的ACL记录中的标记限定符。
　　仅仅当ACE的标记类型为ACL_USER或ACL_GROUP时，标记限定符才有意义。
　　函数acl_get_perset()和acl_set_permset()则可分别用来获取和修改ACE中的权限集合。

创建和删除ACE

　　acl_create_entry()函数用来在现有的某一ACL中新建一条记录，该函数会将一个新建的ACE句柄返回到由其第二个参数所指定的内存位置。 

acl\_entry\_t entry;
status = acl\_create\_entry(&acl,&entry);

　　从ACL中删除一条ACE。 

status = acl\_delete\_entry(acl,entry);

更新文件的ACL

　　acl_set_file()函数的作用和acl_get_file()相反，将使用驻留在内存中的ACL内容来更新磁盘上的ACL。
　　
status = acl_set_file(pathname,type,acl);

ACL在内存和文本格式之间的转换

acl = acl_from_textg(acl_string);
str = acl_to_text(acl,&len);

ACL API中的其他函数

　　alc_calc_mask(acl)函数用来计算并设置内存ACL中ACL_MASK的权限；
