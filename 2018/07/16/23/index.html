进程的终止

Posted on 2018-07-16

进程的终止

　　进程的终止有两种方式，其一异常终止，由接收到一信号引发，默认动作是终止当前进程，并产生核心转储。其二是通过exit()系统调用正常终止。

#include <unistd.h>
void _exit(int status);

　　exit的参数定义了进程的终止状态，父进程可调用wait()获取该状态，虽然将其定义为int类型，但仅有8位可为父进程所用，调用_exi()的函数总会成功终止。（_exit()从不返回）
　　程序一般不会直接调用_exit(),而是调用库函数exit()，它会在调用_exit()前执行各种动作。 

#include <stdlib.h>
void _exit(int status);

　　exit()会执行以下动作：
　　　　1.调用退出处理程序（通过atexit()和on_exit()注册的函数），其执行顺序与注册顺序相反。
　　　　2.刷新stdio流缓冲区
　　　　3.使用由status提供的值执行_exit()系统调用。
　　程序终止的另一种方法就是从main()函数中返回（return），或者一直执行到main()函数的结尾。执行return n 等同于执行对exit(n)的调用，调用main()的运行时函数会将main()的返回值作为exit()的参数。
　　执行时未指定返回值的return，或是无声无息的执行到main()函数结尾同样会导致main()的调用者执行exit()函数。

退出处理程序

　　退出处理程序就是程序设计者提供的函数，可在进程生命周期的任意时点注册，并在该进程调用exit()正常终止时自动执行。如果程序直接调用_exit()或者因信号而异常终止，则不会调用退出处理程序。 

#include <stdlib.h>
int atexit(void (*func)(void));

　　函数atexit()将func加到一个函数列表中，进程终止时会调用该函数列表的所有函数。先注册的程序执行更为基本的清理动作，故所有程序的执行顺序与注册顺序相反。
　　atexit()注册的退出处理程序收到两个限制：
　　　　1.退出处理程序在执行时无法获取传递给exit()的状态。
　　　　2.无法给退出处理程序制定参数。
　　on_exit()可以解除以上限制。其参数func是一个指针，指向以下类型的函数：

void
func(int status,void *arg)
{
    /*perform cleanup actions*/
}

　　提供给exit()的status参数和注册时提供给on_exit()的一份arg参数拷贝。
　　使用atexit()和on_exit()注册的退出处理程序（相关函数）位于同一函数列表。

fork(),stdio缓冲区以及_exit()之间的交互

　　进程是在进程的用户空间内内存中维护stdio缓冲区的，因此，通过fork()创建子进程时，会复制这些缓冲区。有时候可能在子进程退出时导致重复输出等动作。解决方案：
　　　　1.在调用fork()之前使用函数fflush()刷新stdio缓冲区，也可以使用setvbuf()和setbuf()来关闭stdio流的缓冲功能。
　　　　2.子进程可以调用_exit()而不是exit()，以便不在刷新stdio缓冲区。一般父进程通过exit()终止，其他进程通过_exit()终止。
