目录和链接

目录和硬链接

　　在文件系统中，目录的存储方式类似与普通文件，目录和普通文件的区别有2。
　　　　1.在其i-node条目中，会将目录标记为一种不同的文件类型。
　　　　2.目录是将特殊组织形成的文件，本质上是一个表格，包括文件名和i-node编号。
　　　　在大多数的原生的Linux文件系统上，文件名可以长达255个字符。
　　　　进程虽然可以通过read()打开一个目录，但是进程不能使用read()读目录中内容，同样进程不能通过write()函数来改变一个目录的内容。仅能借助于open()，link(),mkdir()，symlin(),unlink()和rmdir()之类的系统调用。
　　　　i-node节点的编号开始于1,如果i-node字段值为0，则表示该节点尚未使用，i-node 1 用来记录文件系统中的坏块。文件系统的根目录总是存储在i-node 2中。
　　　　每个硬链接均指向i-node节点，所有原生的UNIX文件系统均支持硬链接，仅当指向i-node的链接计数为0时，将释放该i-node节点和数据块。
　　　　硬链接的限制：
　　　　　　1.i-node节点编号在一个文件系统中唯一标识一个i-node节点，因此所有硬链接必须与其指代的文件驻留在一个文件系统中。
　　　　　　2.不能为目录创建硬链接，避免出现使程序陷入混乱的链接环路 。

符号软链接

　　符号链接也称为软链接，属于特殊的文件类型，其数据是指另一文件的名称。
　　符号链接的内容可以是绝对路径，也可以是相对路径。解释相对路径是以符号链接本身的位置为参考点。
　　符号连接的地位不如硬链接，文件的链接计数中并未将符号链接计算在内，移除符号链接的文件名内容时，符号链接将继续存在，但是无法解引用操作，也将此链接成为悬空链接。
　　因为符号链接指代一个文件名，而非i-node编号，可以符号链接来链接不同文件系统中的文件，也可以为目录创建符号链接，诸如find和tar 之类的工具有能力识别软链接和符号链接之间的差异，在默认情况下不会对符号链接进行解引用，避免因使用符号链接而陷入环路。
　　系统调用对符号链接的处理：对路径中的目录部分的符号链接总是进行解引用处理，对路径中的文件部分，则取决于路径名所传入的系统调用

创建和移除硬链接：link()和unlink()

#include <unistd.h>
int link(const char *oldpath,const char *newpath);//成功返回0，失败返回-1

　　若oldpath提供了一个已存在文件的路径名，则系统调用link()将以newpath参数指定的路径名创建一个新链接，若newpath指定的路径名已经存在，则产生一个错误。
　　link()系统调用不会对oldpath进行解引用，如果oldpath属于符号链接，则会将newpath创建指向相同符号链接文件的全新硬链接。鉴于系统之间的差异，应避免将oldpath参数指定为符号链接，以保证程序的可移植性。 

#include <unistd.h>
int unlink(const char *pathname);//成功返回0，失败返回-1

　　unlink()系统调用将移除一个链接（删除一个文件名），如果该链接是指向文件的最后一个链接，还将移除文件本身。若pathname中指定的链接不存在，则unlink()调用失败，并将errno设置为ENOENT。
　　unlink()不能移除一个目录，要用rmdir()或者remove()。
　　unlink()系统调用不会对符号链接进行解引用操作，若pathname为符号链接，则移除链接本身，而非链接指向的名称。

仅当关闭所有的文件描述符时，方可删除一个已打开的文件

　　内核除了为每个i-node节点维护链接计数之外，还对文件的打开文件描述计数。当移除指向文件的最后一个链接时，如果仍有进程持有指代该文件的打开文件描述符，那么系统在关闭所有此类描述符之前，实际上将不会删除该文件。

更改文件名:rename()

　　借助于raname()系统调用可以重命名文件，也可以将文件移动到同一个文件系统的另一个目录。 

#include <stdio.h>
int rename(const char*oldpath,const char *newpath);//成功返回0，失败返回-1

　　调用会将现有的一个路径名oldpath重命名为newpath参数指定的路径名。
　　rename()调用仅操作目录条目，不移动文件数据，改名既不影响指向该文件的其他硬链接，也不影响持有该文件打开描述符的任何进程。
　　1.若newpath已经存在，则将其覆盖。
　　2.若newpath与oldpath指向同一文件，则不发生变化（调用成功）

使用符号链接 ：symlink()和readlink()

　　symlink()系统调用会针对由filepath所指定的路径名创建一个新的符号链接-linkpath，想要移除符号链接，需使用unlink调用。 

#include <unistd.h>
int symlink(const char *filepath,const char *linkpath); //return 0 on success,or -1 on error

　　若linkpath中指定的路径名已经存在,则调用失败(且讲errno设置为EEXIST)。由filepath指定的路径名可以是绝对路径，也可以是相对路径。
　　由filepath所命名的文件或目录在调用时无需存在，即便当时已经存在，也无法阻止后来将其删除。这时，linkpath成为“悬空链接”，其他系统调用试图对其解引用时都将出错（通用的错误号为ENOENT）。
　　如果指定一符号链接作为open()调用的pathname参数，那么将打开链接指向的文件。 

#include <unistd.h>
ssize_t readlink(const char *pathname,char *buffer,size_t bufsiz);

　　该函数将符号链接字符串的一个副本置于buffer指向的字符数组中。
　　由于buffer尾部并未放置终止字符串，故而也无法分辨readlink()所返回的字符串到底是经过截断处理，还是恰好将buffer填满。

创建和移除目录：mkdir()和rmdir()

#include <sys/stat.h>
int mkdir(const char *pathname,mode_t mode);

　　pathname参数指定了新目录的路径名，该路径名可以是相对路径，也可以是绝对路径,如果具有该路径名的文件已经存在，则调用失败并将errno置为EEXIST。
　　mode参数指定了新目录的权限，对该位掩码值的既定方式既可以与open()调用相同，既定的mode值还将于进程掩码值相与，另外set-user-ID始终处于关闭状态，因为该位对目录而言毫无意义。
　　如果mdoe中设置了粘滞位(S_ISVTX)，那么将对新目录设置该权限。
　　调用中还会忽略mode中设置的set-group-ID位，相反，如果对父目录设置了set-group-ID位，将导致目录中新建文件的组ID取自目录组ID，而非进程组ID。
　　rmdir()系统调用讲移除由pathname指定的目录，该目录可以是绝对路径名，也可以是相对路径名。 

#include <unistd.h>
int mkdir(const char *pathname);

　　如果要是rmdir()调用成功，要删除的目录必须为空，如果pathname的最后一部分为符号链接，rmdir()调用将不会对其进行解引用操作，并返回错误，并将errno置为ENOTDIR。

移除一个文件或目录：remove()

　　remove()库函数移除一个文件或一个空目录。 

#include <stdio.h>
int remove(const char *pathname);

　　pathname是一个文件，remove去调用unlink,如果pathname是一个符号链接，remove会移除符号本身，而非链接指向的文件。
　　remove属于C标准库函数。

读目录 ：opendir()和readdir()

　　opendir()函数打开一个目录，并返回指向该目录的句柄，供后续调用使用。 

#include <dirent.h>
DIR *opendir(const char*dirpath);

　　opendir()函数打开有dirpath制定的目录，并返回指向DIR类型结构的指针。一旦从opendir()返回，则将目录流指向目录列表的首条记录。 

#include <dirent.h>
DIR *fdopendir(int fd);

　　提供fdopen()函数，用意是帮助应用程序免受各种竞态条件的困扰。 

#include <dirent.h>
struct dirent *readdir(DIR *dirp);

　　每调用一次，就会从dirp所指代的目录流中读取下一个目录条目，并返回一枚指针，指向经静态分配而得的dirent类型结构，每次readdir()都会覆盖该结构。 

struct dirent{
 　　ino_t ino;//file i-node number
 　　char d_name[];//Null-terminate name of file
 　　}

　　调用lstat()（或者stat()，如果应对符号链接解引用时)可获得d_name所指向文件的更多信息，其中路径名是由之前调用oepndir()时指定的dirpath参数与”/“参数以及d_name字段的返回值拼接而成。
　　rewinddir( DIR *dirp)函数可以讲目录流回移到起点，以便对readdir() 的下一次调用将从目录的第一个文件开始。
　　closedir()函数将由dirp指代，处于打开状态的文件流关闭，同时释放流所使用的资源。 　　

目录流与文件描述符

　　有一个目录流，就有一个文件描述符与之关联。dirfd()函数返回与dirp目录流相关联的文件描述符。 

#include <dirent.h>
int dird(DIR *dirp);

文件树遍历：nftw()

　　nftw()函数允许程序对整个目录子树进程递归遍历，并位子树中的每个文件执行某些操作。 　　

进程的当前工作目录

　　进程的当前工作目录定义了进程解析相对路径的起点，新进程的当前工作目录继承自其父进程。

#include <unistd.h>
char *getcwd(char *cwdbuf,size_t size);

　　getcwd()函数将内含当前工作目录绝对路径的字符串(包括结尾字符串)置于cwdbuf指向的已分配缓冲区中调用者必须为cwdbuf缓冲区分配至少sizeg字节的空间。(通常cwdbuf的大小与PATH_MAX常量相当)
　　只要具有合适的权限（大致要求就是进程属主或者具有CAP_SYS_PTRACE能力）就能通过读取（readlink()）linux专有符号链接/proc/PID/cwd的内容来确定任何进程的当前工作目录。

改变当前工作目录

　　chdir()系统调用将调用进程的当前工作目录改变为有 pathname指定的相对或绝对路径名（如果属于符号链接，还会对其接触引用）int chdir(const char *pathname) 

#Include <fcntl.h>
int openat(itn dirfd,const char *pathname,int flags,..);

　　open()系统调用类似于传统的open()系统调用，只是添加了一个dirfd参数，其作用如下：
　　　　1.如果pathname中为一相对路径名，那么对其解释则以打开文件描述符dirfd所指向的目录位参照点，而非进程的当前工作目录。
　　　　2.如果pathname中为一个相对路径名，切dirfd所含位特殊值AT_FDCWD，那么对pathname的解释则是现对于进程的当前工作目录而言。
　　　　3.如果pathname中为绝对路径，那么将忽略dirfd参数。

改变进程的根目录

　　每个进程都有个根目录，该根目录是解释进程绝对路径的起点，默认情况下就是文件系统的真实根目录，特权级进程如CAP_SYS_CHROOT进程可以通过chroot()系统调用做到这点。 

#include <unistd.h>
int chroot(const char *pathname);

　　chroot()系统调用将进程的根目录由pathname指定的目录(如果pathname是符号链接，还将对其解引用)，自此，所有绝对路径名的解释都将以文件系统的这一位置作为起点。

解析路径名：realpath()

　　realpath()库函数对pathname中所有符号链接一一解除引用，并解析其中所有对/.和/..的引用，从而生成一个以空字符串结尾的字符串，内含相应的绝对路径名。 

#include <stdlib.h>
char *realpath(const char *pathname,char * resloved_path);//生成的字符串将置于一个多达PATH\_MAX字节的缓冲区中。
