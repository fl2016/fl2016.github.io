信号

Posted on 2018-07-14

信号

　　信号是事件发生时对进程的通知机制，有时也称之为软件终端。
　　信号分为2大类，第一组用于进程向内核通知事件，构成所谓传统或者标准信号，linux中标准信号的编号范围为1-31。　
　
　　第二组称之为实时信号。　　
　　信号因某些事件产生，信号喊声后，会于稍后被传递给某一个进程，而进程也会采取某些措施来响应信号，在产生和到达期间，信号处于等待（pending）状态。
　　一旦内核接下来要调度该进程的运行，等待信号会马上送达，或者如果进程正在运行，则会立即传递信号，有时候需要将确保一段代码不会位传递来的信号终端，可将信号添加到进程的掩码之中，目前会阻塞该组信号的到达。如果所产生的信号属于阻塞之列，那么信号将保持等待状态，直至稍后对其接触阻塞，即从信号掩码中移除，进程可使用各种系统调用对其信号掩码添加和移除信号。
　　信号到达之后，进程视具体信号执行如下默认动作之一。
　　　　1.忽略信号：就是说内核将信号丢弃，信号对进程没有产生任何影响。
　　　　2.终止进程：有时进程是异常终止。
　　　　3.停止进程：暂停进程的运行。
　　　　4.与之前暂停后再度恢复进程的运行。
　　除了根据特定信号采取默认行为之外，程序也能改变信号到达时的响应行为，也将称之对信号的处置设置。程序将对信号的处置设置为以下之一：
　　　　1.采取默认行为，这适用于撤销之前对信号处置的修改，恢复默认处置的场景。
　　　　2.忽略信号。这适用于默认行为为终止进程的信号。
　　　　3.执行信号处理器程序。

信号默认类型和默认行为

　　SIGABRT 当进程调用abort()函数时，系统向进程发送该信号，默认情况下，该信号会终止进程，并产生核心转储文件。
　　SIGLRM 经调用alarm()或setitimer()而设置的实时定时器一旦到期，内核将会产生该信号，实时定时器是根据挂钟事件进行计时的。

改变信号量：signal()

　　UNIX系统提供了两种方法改变信号处置：signal()和sigaction()。本节描述的是signal()系统调用，是设置信号处置的原始API。所提供的接口比sigaction()简单。但是sigaction()具有signal()所不具备的功能。 

#include <signal.h>
void (*signal(int sig,void(*handler)（int))(int);

　　signal 函数第一个参数sig,标识希望修改处置的信号编号，第二个参数handler,则标识信号到达时所调用的函数的地址。
　　信号处理器函数一般具有以下形式：

void handle(int sig)
{
    /*code for the handler*/
}

　　signal()的返回值是对之前的信号处置，像handler参数一样，是一枚指针，指向一个整形参数且无返回值的函数。

发送信号kill()

　　与shell的kill命令类似，一个进程能够使用kill系统调用向另一个进程发送信号。 

#include <signal.h>
int kill(pid_t pid,int sig); // return 0 on success,or -1 on error。

　　pid参数标识一个或者多个目标进程，而sig则指定了要发送的信号。

  • 如果pid大于0，那么会发送信号给由pid指定的进程。
  • 如果pid等于0，那么会发送信号给与调用进程同组的每个进程，包括调用进程本身。
  • 如果pid小于-1，那么信号的发送范围是：向进程组内的所有组id等于该pid绝对值的所有下属进程发送信号，

  • 如果pid等于-1，调用进程有权讲信号发送到每个目标进程，除去init(进程id是1)和调用进程本身，该信号发送方式也称之为广播信号。

    检查进程的存在

    　　将kill()的系统调用的sig参数指定为0，则无信号发送，如果信号发送失败且errno为ESRCH,则表明目标进程不存在，如果调用失败，且errno为EPERM则表示进程存在。
    　　检查进程是否正在运行：
    　　　1.wait()系统调用。
    　　　2.信号量和排他文件锁。
    　　　3.诸如管道和FIFO之类的IPC通道。可对监控目标进程进行设置，令其在自身生命周期内持有对通道进行写操作的打开文件描述符。同时，令监控进程持有对通道进行读操作的打开文件描述符，且当通道写入端关闭时，即可获知监控目标进程已经终止。
    　　　4./proc/PID接口

    发送信号的其他方式：raise()和killpg()

    　　有时候进程需要向自身发送信号，raise()函数就执行了这一任务。

    #include <signal.h>
    int raise(int sig);// return 0 on success

　　在单线程程序中，调用raise()相当于kill()的如下调用，kill(getpid(),sig);
　　支持线程的系统会将rasie(sig)实现为：pthread_kill(pthread_self(),sig)

显示信号描述

　　每个信号都有一串预支相关的可打印说明，这些smia描述位于数组sys_siglist中。 

#include <signal.h>
extern const char *const sys_siglist[];
#include <string.h>
char strsignal(int sig);

　　strsigna()函数对sig参数进行边界检查，返回一枚指针，指向针对该信号的可打印描述字符串，或是当信号编号无效时指向错误字符串。（strsignal()函数会在sig无效时返回空值。）
　　除去边界检查之外，strsignal()函数较之于直接引用sys_siglist数组的另一优势是对本地设置敏感，所以显示信号描述时会使用本地语言。
　　psignal()函数所示为msg参数所指定的字符串，后面有一个冒号，随后是对应于sig的信号描述，和strsignal()一样，psignal()函数也对本地设置敏感。 

#include <signal.h>
void psignal(int sig,const char *sig);

信号集

　　许多信号相关的系统调用都需要能表示一组不同的信号。sigaction()和sigprocmask()允许程序指定一组将由进程阻塞的信号，sigpending()则返回一组目前正在等待送达给一进程的信号。
　　多个信号可使用一个称之为信号集的数据结构来表示，其系统数据类型为sigset_t。
　　segemptyset()函数初始化一个未包含任何成员的信号集，sigfillset()函数则初始化一个信号集，使其包含所有信号。 

#include <signal.h>
int sigemptyset(sigset_t *set);
int sigfillset(sigset_t *set);

　　必须使用sigemptyset()或者sigfillset()来初始化信号集。
　　信号集初始化之后，使用以下两个函数来向集合中添加或者删除信号。 

#include <signal.h>
int sigaddset(sigset_t *set,int sig);
int sigdelset(sigset_t *set,int sig);

　　setismember()函数用来测试sig是否是信号集set的成员。 

#include <signal.h>
int sigismember(const sigset_t *set,int sig);

信号掩码

　　内核会为每个进程维护一个信号掩码，即一组信号，并将阻塞对该进程的传递（信号掩码实际属于线程属性，在多线程进程中，每个线程都可使用pthread_sigmask()函数来独立检查和修改其信号掩码。）
　　向信号掩码中添加一个信号的几种方式：

  • 调用信号处理器程序时，将引发调用的信号自动添加到信号掩码中。
  • 使用sigaction()函数建立信号处理器程序时，可以指定一组额外信号，当调用该信号处理器程序时会将其阻塞。

  • 使用sigprocmask()系统调用，随时可以显式向信号掩码中添加或移除信号。 

    #include <signal.h>
int sigprocmask(int how,const sigset_t *set,sigset_t *oldset);

　　使用sigprocmask()函数既可以修改进程的信号掩码，又可获取现有掩码。

处于等待状态的信号

　　如果某进程接受了一个该进程正在阻塞的信号，那么会将该信号添加到进程的等待信号之中，如果之后解除了对该信号的锁定，会随之将信号传递给该进程。 

#include <signal.h>
int sigspending(sigset_t *set);

　　suspending()系统调用为调用进程返回处于等待状态的信号集，并将其置于set指向的sigset_t结构中。

不对信号进行排队处理

　　等待信号集只是一个掩码，仅表明一个信号是否发生，未表明信号发生的次数，即如果同同一信号在阻塞状态下产生多次，那么会将该信号记录在等待信号集中，并在解除阻塞之后传递一个。

改变信号处置：sigaction()

　　sigaction()系统调用比之signal()更具灵活性，可移植性更佳。 

#include <signal.h>
int sigaction(int sig,struct sigaction *act,struct sigaction *oldact);

　　sig是想要获取或者改变的信号编号，该参数可以除去SIGKILL和SIGSTOP之外的任何信号。
　　act参数是一个指针，指向描述信号新处置的数据结构，如果仅仅对信号的现有处置感兴趣，那么可将该参数指定为NULL。oldact参数是指向同一结构类型的指针，用来返回之前信号处置相关的信息，如果无意读取该信息，则可将该参数指定为NULL，act和oldact所指向的结构类型如下：

struct sigaction{
    void (*sa_handler) int ;/*access of handler*/
    sigset_t samask;/*signals blocked during handler invocation*/
    int sa_flags;/*flags controlling handler innovation*/
    void (*sa_restorer)(void);/*not for application use*/
  • sg_handler 字段对应于signal()函数的handler参数，其所指定的值为信号处理器的地址，或者是常量SIG_IGN,SIG_DFL之一。仅当sa_handler的取值为信号处理器地址时，才会对sa_mask和sa_flags字段加以处理。
  • sa_mask定义了一组信号，在调用由sa_handler所定义的信号处理器程序时将阻塞该组信号，当调用信号处理器之前，将该组信号中当前未处于进程掩码之列的任何信号自动添加到进程掩码中，直到信号处理器函数返回。这样信号处理器函数不会递归中断自己，由于不会对阻塞的信号进行排队处理，无论在阻塞过程中产生多少同一信号，稍后对信号的传递将会是一次性的。

  • sa_flags 字段将是一个位掩码，指定用于控制信号处理时的各种选项。

    等待信号:pause()

    　　调用pause()将暂停进程的执行，直至信号处理器函数中断该调用为止，或者直至一个未处理的信号终止进程为止。

    #include <unistd.h>
    int pause(void);

　　处理信号时，pause()遭到中断，并总是返回-1，并将errno置为EINTR.
